Queue(Data Structure)

FIFO

First in, First out Elements are added at the REAR, and removed from FRONT.

Linear Array Representation

Overflow & Underflow

Before insertion, check for overflow(If queue is full). If REAR = MAX - 1, Before deletion, check for underflow(If queue is empty). If FRONT = -1, and REAR = -1

Linked List Representation of Queues

If FRONT = REAR = NULL, The queue is empty.

Insert

Step 1. Allocate memory for new node as PTR Step 2. Set PTR->data = val Step 3. If FRONT = NULL // Empty SET FRONT = REAR = PTR SET FRONT -> NEXT = REAR -> NEXT = NULL ELSE SET REAR-> NEXT = PTR SET REAR = PTR SET REAR->NEXT = NULL

Delete

Step 1. If FRONT = NULL write underflow Step 2. SET PTR = FRONT // Essentially, Step 3. SET FRONT = FRONT -> NExT Step 4. FREE PTR

Circular Queue

A drawback of linear queue is that once you insert certain amount of elements, you will not be able to insert any more elements even if there are remaining free memory space due to delete operation.

Insertion in Circular Queue

Check three conditions:

  1. If FRONT = 0 and REAR = MAX - 1, queue is full
  2. If REAR != MAX - 1, queue is no full, increment rear as linear queue
  3. If FRONT != 0 and REAR = MAX-1, queue is not full, go to front and REAR = 0, insert as linear queue

Deletion in Circular Queue

  1. If FRONT = -1, no elements underflow
  2. If Queue is not empty, FRONT = REAR, delete element at front, and set front and rear to -1
  3. if queue is not empty, and FROnt = max - 1, after deletion, front is set to 0.

Dequeue

A double ended queue, can insert and delete from left and right

Priority Queue

Three elements: Data, Priority, and Next. Insert if priority is higher or equal to. Delete at front
