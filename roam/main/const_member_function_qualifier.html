Const Member Function Qualifier

1. The "Read-Only" Guarantee

By adding const there, you are promising the compiler that this function will not modify any member variables of the object.

Inside the function:

You cannot change the value of x or y.

You cannot call other member functions that are not marked as const.

2. Compatibility with const Objects

This is the most practical reason for using it. If you have a const instance of your struct, you can only call member functions that are marked with that second const. C++

void check(const P& a, const P& b) { if (a = b) { // This works ONLY if operator= is a const member function // … } }

If you omitted the second const, the compiler would throw an error in the snippet above. Even though operator== doesn't actually change anything, the compiler assumes a non-const function might change the object, so it forbids calling it on a const object like a.

  1. The this Pointer Transformation

Under the hood, every member function has a hidden parameter called this.

In a normal member function of P, the type of this is P* const (a constant pointer to a mutable object).

In a const member function of P, the type of this is const P* const (a constant pointer to a constant object).

In competitive programming or system design, it is best practice to mark all "getter" or comparison operators as const. This prevents accidental bugs where you might unintentionally modify a value during a simple check.
