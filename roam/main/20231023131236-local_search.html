Local Search

Local search means the search space is local. Only the current(and possibly some other states) are kept and used for search. When would we do local search? when the global state i.e. path is irrelevant. Only the passing the goal test(finding the goal node) is important.

Examples where local search works is when you are trying to find some configuration that satisfies given constraints, or when you wish to find optimal configuration.

Iterative improvement

Keep single state and improve it

Let us use the n-queens as an example for local search The goal is to have n queens with no conflicts States is the board, and the location of the queens, Actions, is to move a queen. Heuristics is the number of conflicts.

Hill Climbing

n-queens- choose successor state ( the state with lowest conflict) If the value of the successor is lower than or equal to the current value, you are in local maxima, which may be global, but regardless you will not be able to find better solution, so you return with current value you may use sideways moves to escape the said local maxima, but it may not always work

Simulated annealing

The rationale behind this method is that since hill climbing finds local(greedy) optima, given that search space is dense and smooth, It may suffice to attempt search on multiple points, even if the points are random.
